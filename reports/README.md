# Reports Directory

This directory is designated for storing analytical artifacts generated by the AI Agent and post-incident documentation.

## Types of Reports

- **`incident_summaries/`**: High-level summaries of detected incidents, showing timelines, impact, and initial categorization.
- **`forensic_reports/`**: Detailed log parsing and step-by-step trace analysis of suspicious activity.
- **`remediation_actions/`**: Logs of automated steps taken to contain the threat (e.g., ports closed, accounts isolated) along with manual action recommendations for security analysts.
- **`post_mortems/`**: Standardized post-incident review documents detailing lessons learned, playbook improvements, and timeline reviews.

## Report Generation

The AI Agent automatically saves markdown formatted summaries of its findings here upon completing an investigation. These reports follow a standardized JSON/Markdown schema to facilitate integration into Ticketing Systems (e.g., Jira, ServiceNow).
